L0AB 双缓冲 原理详解

Cube 做矩阵乘要的操作数在 L0A/L0B —— 为什么这两块各开「两份」轮流用，让 MTE1 的上料和 Cube 的算并行、互不干等

本文假设读者已熟悉昇腾的基本盘：GM / L1 / L0A / L0B / L0C 各级内存，Cube 单元做矩阵乘、操作数得先在 L0 里，数据靠 MTE 在各级之间搬。

要讲清楚的是：L0A 和 L0B 为什么各开「两份」轮流用（双缓冲 / 乒乓）。这是一招很通用的「多备几份、轮流用」思路，本文聚焦它在 L0A/L0B（即 L1→L0 这一跳）上的用法。只需会分奇偶即可跟下来。
（范围说明：在这个算子里，这套 L0A/L0B 双缓冲是手写在 PV 汇总阶段的；QK 打分是另一个矩阵乘原语一步算完的，它内部怎么摆放 L0 属于原语内部、不在本文范围。所以下文一律以 PV 为例。）

0先说结论：这两份缓冲到底在忙什么

Cube 单元做一次矩阵乘，要两个操作数：左矩阵放在 L0A、右矩阵放在 L0B，算出的结果进 L0C。可这两个操作数原本在 L1 里，得先靠 MTE1 上料（L1→L0）到 L0A/L0B，Cube 才能开算。问题还是那个老问题：

如果 L0A、L0B 各只有一份，那么上料（MTE1 往里写）和算（Cube 从里读）就没法同时进行 —— 同一块缓冲，正被 Cube 读的时候不能被 MTE1 覆盖。于是两边被迫轮流，谁在忙、另一个就干等。

L0AB 双缓冲就是来消掉这个干等的：把 L0A、L0B 各开「两份」，轮流用 —— Cube 就着这一份算的时候，MTE1 同时往另一份上下一批料。两件事就此并行，谁也不必停下来等谁 —— 总时间取决于两者里较慢的那件，而不是两件相加。下面一步步讲清楚。

1L0A/L0B 在哪、装什么

先把这一段通路和各部件的角色理清楚：

L1──MTE1 上料──▶ L0A/L0B ──Cube 算──▶ L0C
  • L0A 放矩阵乘的左操作数，L0B 放右操作数。Cube 一次 mma 同时读这两块，把结果累加进 L0C。
  • 本文这套 L0A/L0B 双缓冲用在 PV 汇总阶段：L0A 放 P（softmax 权重），L0B 放 V 的一个小片。（QK 打分由一个矩阵乘原语一步算完，它内部怎么用 L0 不在本文范围。）
  • 双缓冲落在 L1→L0（MTE1）这一跳 —— 数据从 GM 一路往里搬（GM→L1→L0），这是最靠里的一段。

📌 数据一路往里搬：GM →(MTE2)→ L1 →(MTE1)→ L0 →(Cube)→ L0C。每一跳的「搬」和「用」之间都能做双缓冲；本文只讲最后那跳 L1→L0（MTE1）上的这一处 —— L0A/L0B 各开 2 份。
（提醒：L1→L0 是片上搬运、比 GM→L1（片外显存）快得多，所以这层双缓冲省下的时间有限、是个小优化——PV 每个任务也就 4 次这样的 mma；真正的搬运大头在更外面那跳 GM→L1。）

2各只一份：上料（MTE1）和算（Cube）互相干等

先看最朴素的做法：L0A、L0B 各只开一份。那流程只能是：

  1. MTE1 把这一批操作数从 L1 上料到 L0A / L0B；
  2. 上完料，Cube 才能读它、算这一次 mma；
  3. 算完，MTE1 才敢上下一批（只有一份，没算完就上料会把还在用的操作数覆盖掉）；
  4. ……如此循环。

画成时间线：MTE1 和 Cube 从来没有同时在干活。上料时 Cube 空等，算时 MTE1 空等。

3各开两份：一边算一边上料（乒乓 / 双缓冲）

改进只要一句话：L0A、L0B 各多开一份。有了两份（记作第 0 份、第 1 份），就能这样安排：

Cube 就着第 0 份算的时候，MTE1 同时往第 1 份上下一批料。等第 0 份算完，第 1 份也刚好上好料 —— Cube 扭头就着第 1 份接着算，MTE1 则回头往第 0 份上再下一批。两份交替上阵、轮流坐庄，这就是 双缓冲（乒乓）。

4一个关键点：L0A 和 L0B 是「成对」轮换的

这里有个容易忽略的细节：一次矩阵乘要同时用到左操作数（L0A）和右操作数（L0B）。所以这两块缓冲的「第 0 份」得配对使用、「第 1 份」也配对使用；它俩一起在 0 / 1 之间翻，不各翻各的。

可以把「一份 L0A + 同号的一份 L0B」看成一套配好的料（下面叫「第 0 套 / 第 1 套」）：

📐 「两份」的具体形状：L0A 每份 [64,128]（放 softmax 权重 P：64 查询头 × 128 key），L0B 每份 [128,128]（放 V 的一个小片：128 key × 128 维）；各开 2 份，合起来就是 [2,64,128] 和 [2,128,128]。

5为什么 2 份就够

这里自然有个问题：双缓冲为什么是 2 份，而不是 3 份、4 份？

因为 L0 这一层是最标准的「产一个、消费一个」双缓冲：Cube 一次 mma 只吃一套料，算这一套的同时把下一套备好 —— 手上至多两套（一套在算、一套在备），2 份正好一一对应，再多一份也没活干。

那什么时候才需要更多份？只有当某一层需要把好几批料同时摆着一起用（而不是「一次只消费一套」）时，份数才要相应加多。L0 这层没有这种情况 —— 每次 mma 就吃一套，所以标准的 2 份乒乓就够。

6握手信号，以及「开工前预置、收尾排空」

两份轮流用要跑得又快又不出错，靠两块缓冲各自的一对握手信号（flag）。盯住某一份，它的一轮生命是这样的：

两份缓冲各跑这么一个循环，但错开半拍：Cube 在算第 0 份时，第 1 份早已收到过「算完」、正被 MTE1 上料 —— 这就是重叠。

6.1 开工前「预置」一次、收尾「排空」一次

这里有个先有鸡还是先有蛋的小问题：MTE1 第一次上料前，需要先收到「算完」信号才敢写 —— 可循环刚开始，根本还没人算过、没人发过「算完」。怎么破？

办法是：在进主循环之前，先「预置」一次 —— 假装两份都已经「算完」、是空的、可以随便写。这样第一次上料就能顺利开始。对应地，整个循环跑完后，再「排空」一次，把当初预置发出、但没人消费的那两个信号收回来配平。

关键在于：这对「算完」信号是开工前预置一次、收尾排空一次，中间一直复用 —— 每一次 mma 用完随手重新置上，不必在每次 mma 的边界处都重新预置、清理一遍，省事又高效。

🔁 凡是这种「循环里反复用、靠反向信号保护」的缓冲，都要在循环两头各配平一次：进循环前把信号置成「空闲可写」（预置），出循环后把没消费掉的信号等回来（排空）。

7一页速查 + 总结

整张图

三句话总结

  1. L0A/L0B 是喂给 Cube 的操作数缓冲：L0A 放左矩阵、L0B 放右矩阵，Cube 一次 mma 同时读两块、结果进 L0C；操作数由 MTE1 从 L1 上料。
  2. 各开两份是为了重叠：Cube 就着这一份算时，MTE1 往另一份上下一批料，两件事并行、互不干等（双缓冲 / 乒乓），总时间约等于两者里较长的那条、而不是相加。L0A、L0B 的同号份配成一套、一起 0/1 轮换（一次 mma 要左右两个操作数）。
  3. 2 份就够：L0 这层是标准的「产一套、消费一套」双缓冲，一次 mma 只吃一套、手上至多两套，2 份正好错开（多一份也没活干）。靠两个方向的握手信号（上料完 / 算完）保证不出错；那对「算完」信号开工前预置一次、收尾排空一次，中间一直复用。
问题答案
L0A / L0B 装什么？矩阵乘的左 / 右操作数（PV 阶段：L0A 放 P、L0B 放 V 小片）。Cube 读两块算，结果进 L0C。
为什么各开两份？让 MTE1 的上料和 Cube 的算重叠，别让 Cube 干等上料。
两份怎么轮流？按奇偶（0,1,0,1…）乒乓。L0A、L0B 同号配一套、一起翻。
为什么 2 份就够？L0 这层「一次 mma 吃一套」= 标准的产一套、消一套，手上至多两套，2 份正好错开、多一份也没活干。
为什么要预置 / 排空？第一次上料前没人发过「算完」，所以进循环前先预置（假装空、可写）；出循环后把没消费的信号排空配平。整段循环共用这对信号。

L0A/L0B 各多备一份，就让 MTE1 的上料和 Cube 的算稳稳并行 —— 又快的算藏进了较慢的上料背后。这就是 L0AB 双缓冲的全部用意。
