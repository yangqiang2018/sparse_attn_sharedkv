Online Softmax 原理详解

一块一块喂数据、只用一点点内存、绝不溢出，还和「一次性全算」分毫不差 —— 高中生也能读懂

目标读者：高中生也能读懂。只需要你会这几样东西：

  • 加减乘除、求最大值、求和；
  • 知道指数函数 exp(x)（也写成 eˣ）：x 越大它越大，而且永远是正数；
  • 知道对数 ln(x) 是 exp 的反函数（ln(eˣ) = x）。

全文用同一个小例子从头贯穿到尾，每一步都能拿计算器自己验算。

0先说人话：这玩意儿到底在算什么

在 Transformer 的注意力（attention）里，有这么一件事要做：

对于当前这个词（query），去看序列里所有的「键」（key），算出它应该多关注每一个键（这是一组权重，加起来等于 1），然后按这组权重，把每个键对应的「值」（value）加权平均起来，作为输出。

把分数变成「加起来等于 1 的权重」，用的就是 softmax。整件事用一张图就是：

听起来很简单，对吧？问题出在四个字：又长又大。

  • 序列很长（可能几千上万个键），分数一次性放不进高速内存；
  • 分数经过 exp 之后很大，普通的浮点数会溢出（变成 inf，全盘崩溃）。

Online Softmax（在线 softmax） 就是为了解决这两个问题而生的算法：它让我们能一块一块地喂数据进去，只用很小的一点内存，还能完全避免溢出，最后算出的结果和「一次性全算」一模一样，不是近似。

本文就是要把这件事讲透。我们先从最朴素的 softmax 讲起。

1第一步：普通 softmax —— 把一堆分数变成概率

1.1 公式

给定一串分数 s₁, s₂, …, sₙ，softmax 把它们变成一串概率 p₁, p₂, …, pₙ：

pᵢ= exp(sᵢ)exp(s₁) + … + exp(sₙ)

也就是：每个分数先取 exp，再除以所有 exp 的总和。这样得到的 pᵢ 有两个好性质：

  1. 每个都是正数（因为 exp 永远是正的）；
  2. 全部加起来正好等于 1（分子之和 = 分母）。

所以它是一组合法的「权重 / 概率」。

1.2 为什么要用 exp，而不是直接除？

因为 exp 会放大差距：分数高的拿到的权重会多得多，分数低的几乎被忽略。这正是注意力想要的效果 —— 「重点关注最相关的那几个键」。

直觉：exp 就像评委打分时的「赢者通吃」。1 分和 3 分看起来只差 2，但 exp(3)/exp(1) = e² ≈ 7.4 倍。

到这里，「普通 softmax」讲完了。但它有两个致命问题，下面逐个拆解。

2第一个麻烦：exp 会「爆炸」—— 减最大值技巧

2.1 问题：浮点数装不下

我们例子里的分数最大才 5，exp(5)≈148，没事。但真实模型里分数可能是几十、上百。exp(1000) 是个天文数字，计算机的浮点数（float）根本存不下，结果会变成 inf（无穷大），然后 inf / inf = NaN（非数字），整个计算彻底报废。

2.2 解法：先把每个分数都减去「最大值」

诀窍很简单：先找出所有分数里的最大值 m，然后把每个分数都减去 m，再做 softmax。

为什么可以这么做？因为 softmax 是个分数（比值），给分子分母同时乘以同一个常数，结果完全不变。我们选的常数就是 exp(−m)：

pᵢ= exp(sᵢ)Σ exp(sⱼ) = exp(sᵢ)·exp(−m)Σ exp(sⱼ)·exp(−m) = exp(sᵢ−m)Σ exp(sⱼ−m)

数学上 100% 等价，一点没变。 但好处是：

  • 最大的那个分数减完变成 0，exp(0)=1，不会再溢出；
  • 其它分数减完都 ≤ 0，exp 出来都在 0~1 之间，安全。

这一招叫 「safe softmax」（数值稳定的 softmax）。换个角度看，「减最大值」其实就是把整把尺子平移，让最高分落在 0 上：

记住这个数：减完最大值后，分母是 1.5781。后面会反复用到。（它和原始分母 234.204 的关系是 234.204 = exp(5) × 1.5781，因为我们给所有项都乘了 exp(−5)。）

到这里我们解决了「爆炸」。但还有第二个麻烦。

3第二个麻烦：一次看不到全部分数 —— 必须分块

3.1 矛盾点在哪

第 2 节的「减最大值」技巧有个隐含前提：你得先知道最大值 m。而要知道最大值，你必须先把所有分数都看一遍。

可是在真实硬件上（比如昇腾 NPU），事情是这样的：

  • 序列很长，几千个键的分数一次性放不进芯片里那块又小又快的内存（UB / L1）；
  • 所以分数是一块一块（chunk by chunk）算出来的。每块是固定数目的键（比如 64 个）。

于是矛盾来了：

我想减最大值（需要先看完全部），但我又只能一块一块地看（看不到全部）。

怎么办？难道每块都先存起来，等全部算完再回头处理？那样就违背了「省内存」的初衷 —— 序列一长，存不下。

3.2 思路：能不能「边走边更新」？

Online softmax 的回答是：可以！

我不存全部分数。我只在手边维护两个小账本：

  • m：到目前为止见过的最大分数（running max）；
  • d：到目前为止已经累加的分母（running sum，即 exp 之和）。

每来一个新块，我就更新这两个账本。关键技巧是：如果新块里冒出了一个更大的最大值，我就把旧账本「修正」一下，让它和新的最大值对齐。

这个「修正」是整个算法的灵魂。下一节详细讲。

4核心思想：Online Softmax —— 边走边修正

4.1 为什么旧账要「修正」

打个比方 🏀：

你在一个房间里给每个人测身高，但你只想记录「比当前最高的人矮多少」。一开始最高的是小明（180cm），你记下大家相对小明的差值。突然走进来一个 200cm 的巨人 —— 现在「最高」变了！你之前记的所有差值（相对 180）全都过时了，必须统一加上 −20（相对 200 重新算），才能和新人放在一把尺子上比较。

Online softmax 里也是一样。我们存的分母 d 是 Σ exp(分数 − 当前最大值)。一旦最大值从 m_旧 变成 m_新（更大），旧的每一项 exp(分数 − m_旧) 都要乘上一个修正因子：

exp(分数 − m_新)  = exp(分数 − m_旧)旧账本里存的 × exp(m_旧 − m_新)修正因子（≤ 1，打折扣）

所以只要把整个旧账本 d 乘以 exp(m_旧 − m_新) 这一个数，就等于把里面每一项都重新对齐到新最大值了。一次乘法，搞定全部历史。

注意 m_旧 ≤ m_新，所以 m_旧 − m_新 ≤ 0，修正因子 exp(…) ≤ 1。它是一个「打折扣」的系数 —— 把旧账本缩小，因为我们换了一把更大的尺子。

4.2 算法（先只看分母 d）

把例子的 6 个键分成两块，每块 3 个：

  • 第 1 块：键 1,2,3，分数 [1, 3, 2]
  • 第 2 块：键 4,5,6，分数 [5, 4, 0]

初始化：m = −∞（还没见过任何数），d = 0。下面这条「账本流水」一步步走，每一步都标出两个小账本的最新值：

初始化
m = −∞ d = 0
处理第 1 块 ↓
第 1 块 分数 [1, 3, 2]
① 局部最大值 m_块 = 3
② 更新最大值 m_新 = max(−∞, 3) = 3
③ 修正因子 = exp(−∞ − 3) = 0（旧账空，乘 0 没影响）
④ 块内 exp(分数 − 3)：exp(−2), 1, exp(−1) = 0.1353, 1, 0.3679 → 和 = 1.5032
⑤ d = 0 × 0 + 1.5032 = 1.5032
m = 3 d = 1.5032
处理第 2 块 ↓
第 2 块 分数 [5, 4, 0] 最大值变大 ⬆
① 局部最大值 m_块 = 5
② 更新最大值 m_新 = max(3, 5) = 5 ← 变大了！
③ 修正因子 = exp(3 − 5) = exp(−2) = 0.1353
④ 先修正旧账本：d = 1.5032 × 0.1353 = 0.2034
   （把第 1 块的旧分母从「相对 3」重新对齐到「相对 5」）
⑤ 块内 exp(分数 − 5)：1, exp(−1), exp(−5) = 1, 0.3679, 0.0067 → 和 = 1.3746
⑥ d = 0.2034 + 1.3746 = 1.5780
m = 5 d = 1.5780

4.3 见证奇迹的时刻

我们分两块、边走边修正算出来的最终分母是 d = 1.5780。第 2 节里「一次性全算」得到的分母是 1.5781。

这就是 online softmax 的核心承诺：

分块在线计算的结果，和一次性全算的结果完全相同，不是近似。而代价只是多存了 m 和 d 两个小数字，外加每次更新时一次「乘修正因子」。

5注意力不止要概率，还要加权求和 V

到上一节为止，我们只算了分母 d。但注意力真正要的输出是：

输出 O=Σ pᵢ · Vᵢ= Σ exp(分数ᵢ − m) · Vᵢd

也就是用 softmax 权重去加权平均所有的值 Vᵢ。在动手算之前，先分清楚一件容易绕晕的事：谁是标量、谁是向量。

⚠️ 下面的小例子为了能用计算器手算，把每个 Vᵢ 简化成了单个数字（10, 20, …），所以例子里的 O 看起来像一个数。但请记住：真实情况下 Vᵢ 是 512 维向量，O 也是 512 维向量，每一维都独立地、按和本文完全相同的规则更新。

好消息是：分子（加权和）也可以用一模一样的「边走边修正」来算。我们再多维护一个账本：

  • O：到目前为止累加的（未归一化的）加权值之和。

它遵守和 d 完全相同的修正规则：每次最大值变大，就把 O 乘上同一个修正因子 exp(m_旧 − m_新)，然后加上新块的贡献。最后一步才把 O 除以 d，得到归一化的输出。

为什么 O 和 d 用同一个修正因子？因为它们的每一项都含有同样的 exp(分数ᵢ − m)，只是 O 的那一项还多乘了一个 Vᵢ。换尺子（换最大值）时，exp 那部分要打的折扣是一样的，Vᵢ 不受影响。

5.3 一个重要的领悟：最终的概率 pᵢ 其实从没被真正算出来

你可能已经隐约感觉到了：整个过程里，我们从头到尾都没有单独算出任何一个「最终概率」pᵢ。这正是 online softmax（也是 FlashAttention）最精髓的地方。

回忆最终概率的定义：pᵢ = exp(sᵢ − m_最终) / d_最终。它需要两样全局信息：最终的最大值 m_最终 和最终的分母 d_最终，而这两样只有处理完最后一块才知道。所以在中途，我们根本没法算出任何一个真正的 pᵢ。

那中途算的是什么？是临时的、未归一化的权重 p̃ᵢ = exp(sᵢ − m_当前)：

  • 未归一化：没除以 d（因为 d 还没定）；
  • 临时：相对的是当前的运行最大值，不是最终的 —— 一旦遇到更大的最大值，它的贡献就要被修正因子重新打折；
  • 只存一块的量：任何时刻内存里只有一块的权重，完整的注意力概率向量从不存在。

我们拿这些临时权重直接乘 Vᵢ 累加进 O，然后把「除以 d」这一步推迟到最后，只对 O 做一次。于是真正的 pᵢ 始终没有以一个完整数组的形式存在过，但它的效果通过「逐块修正 + 最后归一化」准确地体现在了 O 里。

⚠️ 注意：说「没算出概率」不等于「没算 exp」。每处理一块，确实算了 exp 权重（之后会拿它去和 V 做加权，即 P@V）。只是那是未归一化的临时权重，不是除过 d 的最终概率。

用例子看第 2 个键（s=3, V=20）： 它的「真正概率」是 p₂ = exp(3−5)/1.578 = 0.0858。但全程我们一次都没单独算过它，它的贡献是这样一路「攒」进 O 的：

而 p₂ × V₂ = 0.0858 × 20 = 1.715 —— 结果分毫不差。但你看，那个 0.0858 我们一次都没单独算出来过，它只是隐含在 O 的最终数值里。

结论：online softmax 不是「先把概率都算出来再加权」，而是把 softmax 的归一化与 O 的加权求和融合（fuse）在一起，把归一化推迟到最后一次性完成。 这样就永远不必在内存里存下完整的注意力概率向量 —— 这正是 FlashAttention 这类算法又快又省内存的根本原因。

至此，注意力的完整在线计算就讲完了。我们只用了三个小账本 m, d, O，就把任意长序列的注意力分块、不溢出、零误差地算了出来。

6一个常见变体：sink 种子

前面为了好懂，我们把 m, d, O 初始化成 −∞, 0, 0。但带 attention sink 时，初始化不太一样：

# 用 sink 给在线 softmax 播种
m = sink      # m 初始化为 sink 的分数（不是 −∞）
O = 0         # O 初始化为 0      （这个一样）
d = 1.0       # d 初始化为 1.0    （不是 0！）

m 从一个叫 sink 的分数开始，d 从 1.0 开始。这是什么操作？

6.1 什么是 attention sink

你可以把 sink 理解成一个虚拟的「第 0 号键」，它有两个特点：

  1. 它的分数是一个固定（或可学习）的值，叫 sink；
  2. 它的值向量 V 全是 0。

把它当成普通键塞进 softmax 会发生什么？

  • 它对分母的贡献是 exp(sink − m)。如果一开始就让 m = sink，那么这一项 = exp(0) = 1 —— 这就是为什么 d 初始化成 1.0！
  • 它对分子 O 的贡献是 exp(sink − m) × V₀ = 1 × 0 = 0 —— 所以 O 还是从 0 开始。

换句话说，「种子」就是把这个虚拟的零值键预先放进账本里。并没有真的造一个键，而是直接把它对 m, d, O 的初始贡献写好，省事又高效。

6.2 sink 有什么用

它给了模型一个「什么都不关注」的出口。

普通 softmax 强迫所有权重加起来等于 1 —— 哪怕当前 query 觉得「这些键我一个都不太想理」，它也被迫把 100% 的注意力分摊出去。有了 sink，模型可以把一部分注意力「倒进」这个零值虚拟键里，相当于说「这部分注意力我不用了」。这能让训练更稳定，也是很多大模型里的常见技巧。

第 2 块 [5,4,0] ↓
第 2 块 m_新 = max(3,5) = 5 ⬆，修正因子 = exp(3−5) = 0.1353
修正旧账：d = 1.8711 × 0.1353 = 0.2532；O = 32.390 × 0.1353 = 4.383
加这块：d = 0.2532 + 1.3746 = 1.6278；O = 4.383 + 58.798 = 63.182
m = 5 d = 1.6278 O = 63.182
收尾 ↓
输出 = O / d = 63.182 / 1.6278 = 38.813

对照「一次性全算 + sink」：分母里多了 sink 那一项 exp(2−5)=exp(−3)=0.0498，所以 d = 1.5781 + 0.0498 = 1.6279；分子不变（sink 的值是 0）。输出 = 63.182 / 1.6279 = 38.813。又对上了！ ✅

7顺手的副产品：LSE（log-sum-exp）

在线 softmax 在最后还会顺手多产出一个量，叫 LSE：

lse = ln(d)       # 先取分母 d 的对数
lse = lse + m     # 再加回最大值 m

也就是：

LSE=m + ln(d)

7.1 它是什么

LSE 是 "log-sum-exp" 的缩写，意思是 「所有原始分数取 exp、求和、再取对数」：

LSE=ln( exp(s₁) + exp(s₂) + … + exp(sₙ) )

为什么 m + ln(d) 就等于它？因为我们存的 d 是减完最大值的分母 d = Σ exp(sᵢ − m)，所以一步步推：

m + ln(d) = m + ln( Σ exp(sᵢ − m) )
= ln( exp(m) · Σ exp(sᵢ − m) ) ← 把 m 塞进 ln 里变成 exp(m)
= ln( Σ exp(sᵢ) ) ← exp(m)·exp(sᵢ−m) = exp(sᵢ)
= LSE ✓

妙处：我们手上正好有 m 和 d，所以 LSE 几乎是白送的 —— 一次 ln 加一次加法就出来了，不用重新扫描数据。

7.2 用例子验证

不带 sink：m = 5，d = 1.5781。LSE = 5 + ln(1.5781) = 5 + 0.4563 = 5.4563。直接验算：ln(234.204) = 5.4563（234.204 就是第 1 节那个原始分母）。✅

带 sink：m = 5，d = 1.6279。LSE = 5 + ln(1.6279) = 5 + 0.4873 = 5.4873。✅

7.3 LSE 有什么用

  • 训练反向传播需要它：softmax 的梯度公式里要用到这个归一化常数的对数；
  • 拼接多段注意力时需要它：当一个超长序列被拆到不同核 / 不同时间段分别算，要把各段的部分结果合并成全局结果，靠的就是各段的 LSE 来重新加权（这正是 FlashAttention 能跨块、跨设备拆分的数学基础）。

所以实现里一般总会把 LSE 顺带算出来 —— 它几乎是零成本的（"essentially free"）。

8一页速查：完整伪代码 + 总结

完整伪代码

# ---- 初始化（来自 sink 种子）----
m = sink            # 运行最大值
d = 1.0             # 运行分母（= sink 那一项 exp(0)）
O = 0               # 运行输出（未归一化；sink 的值为 0）

# ---- 一块一块地处理所有键 ----
for 每一块 keys:
    s        = 缩放后的分数        # 被 mask 掉的位置 = −∞（exp 后为 0，不贡献）
    m_块     = max(s)             # 这一块的局部最大值
    m_new    = max(m, m_块)       # 更新运行最大值
    corr     = exp(m − m_new)     # 修正因子（旧账要打的折扣，≤ 1）
    p        = exp(s − m_new)     # 这一块每个 key 的未归一化权重
    d        = d × corr + sum(p)               # 修正旧分母，再加新块
    O        = O × corr + sum(pᵢ × Vᵢ)         # 修正旧输出，再加新块
    m        = m_new

# ---- 收尾 ----
输出  = O / d           # 整个序列处理完，只归一化这一次
LSE   = m + ln(d)       # 顺手产出 log-sum-exp

三句话总结

  1. softmax 要减最大值，否则 exp 溢出 → 用 safe softmax。
  2. 但分块时看不到全局最大值 → 维护运行最大值 m，一旦遇到更大的，就用修正因子 exp(m_旧 − m_新) 把旧的分母 d 和输出 O 统一「打折」对齐。
  3. 结果与一次性全算完全相等（零误差），代价只是几个标量账本 + 每块一次修正乘法；顺带白送一个 LSE = m + ln(d)，供反向传播和跨块合并使用。
